# Internship Selection Filter

Use this file when deciding whether an opportunity is worth pursuing.

## Must-Have Questions

1. Does it connect to grid, BESS, power electronics, EMS, reliability, or energy software?
2. Will I learn technical depth, not just operations?
3. Can I produce a project, report, GitHub repo, or publication?
4. Will it strengthen my Germany/master's profile?
5. Does it expose me to real industrial pain?
6. Does it move me closer to the energy infrastructure company vision?

## Highest-Priority Internship Categories

- BESS analytics
- Battery diagnostics
- Grid technology
- Power electronics
- Inverter testing
- EMS software
- Protection systems
- Embedded energy systems
- Microgrid control
- Industrial reliability analytics

## Medium-Priority Categories

- PV reliability
- Solar manufacturing QA
- Energy data analytics
- EV charging infrastructure
- Smart grid software

## Lower-Priority Categories

- Generic solar installation
- Sales
- Project documentation only
- Repetitive manufacturing without data or project access
- Non-technical renewable business roles

## Avoid Drift

Avoid drifting into:

- Only solar panel manufacturing
- Only PV installation
- Generic web dashboards
- Superficial AI projects
- Pure business without engineering depth
- Trying to build three separate startups
- Overbuilding architecture before solving the real problem
- Using Kafka, CNNs, or LLMs just because they sound advanced
- Ignoring market pain
- Ignoring data availability
- Ignoring physical engineering constraints
