# 90-Day Action Plan

This plan converts the roadmap into near-term execution.

## Operating Principle

The goal for the next 90 days is not to build the full company. The goal is to create credible proof of technical direction:

> Industrial reliability + image/data analytics + disciplined technical writing.

## Days 1-15: Foundation

### Learn

- Solar module manufacturing flow
- EL imaging basics
- Microcracks vs PID
- IV curve basics
- IEC 61215 and IEC 61730 overview

### Build

- Daily internship log folder
- Glossary of manufacturing and reliability terms
- First process-flow diagram

### Output

- One 2-page note: "Solar Module Manufacturing and Reliability Failure Modes"

## Days 16-30: Data and Methods

### Learn

- OpenCV image loading, thresholding, contour detection, morphology
- Basic statistics: correlation, regression, confidence intervals
- How flash-test metrics relate to defects

### Build

- EL image processing notebook using public/sample images if company data is unavailable
- Feature extraction plan
- Data schema for module/cell observations

### Output

- One notebook or script that extracts basic image features
- One technical note: "Quantifying EL-Visible Defects"

## Days 31-45: Research Framing

### Learn

- Literature on EL defect classification
- Literature on microcrack impact on PV module performance
- TOPCon vs PERC reliability considerations

### Build

- Literature matrix
- Research question list
- Draft project proposal

### Output

- 4-6 page research proposal
- Slide deck outline for APS/professor discussion

## Days 46-60: Analysis Prototype

### Learn

- Regression
- ANOVA
- PCA or feature importance
- Measurement uncertainty

### Build

- Defect feature table
- Correlation analysis template
- Draft defect severity index method

### Output

- Prototype analysis report with plots
- Clear list of data needed from APS, only if allowed

## Days 61-75: Business Translation

### Learn

- False rejection vs false acceptance
- Warranty risk
- Energy yield impact
- Yield improvement logic

### Build

- Reliability-to-business impact map
- Simple cost model for defect screening decisions

### Output

- One note: "How EL Defect Analytics Can Reduce Warranty and Yield Loss"

## Days 76-90: Portfolio Package

### Build

- Clean GitHub repository
- Research report draft
- Technical presentation
- Internship learnings summary

### Output

- Portfolio-ready PV reliability analytics project
- Next-internship narrative aimed at BESS, grid, EMS, or power electronics roles

## Weekly Review Questions

- What did I learn that connects to reliability or grid/battery systems?
- What real industrial pain did I observe?
- What non-confidential data could support a project?
- What would make this more technically rigorous?
- What would make this stronger for Germany/master's applications?
