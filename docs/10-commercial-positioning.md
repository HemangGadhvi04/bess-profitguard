# Commercial Positioning

## Product Stance

BESS ProfitGuard should not be positioned as a full live EMS control product in the first version.

The first commercial wedge is:

> Paid BESS / EV charging dispatch audit and battery degradation report.

This is a decision-support product, not an automatic control system.

## What The First Customer Pays For

The customer is not paying for a dashboard. They are paying for an answer:

> Are we using this expensive battery in a profitable way, or are we damaging it for small savings?

The first paid deliverable should be a report that includes:

- data validation
- battery health summary
- stress events
- high-temperature and high-SoC warnings
- equivalent full cycles
- degradation cost estimate
- naive dispatch versus degradation-aware dispatch
- recommended operating schedule
- savings and lifetime-risk summary

## Early Customers

Best first targets:

- solar EPCs adding BESS to C&I projects
- EV charging station operators
- factories with solar + battery systems
- BESS O&M / maintenance companies
- small battery or inverter companies that need analytics/reporting
- energy consultants
- professors or labs with microgrid/BESS data

Avoid starting with large utilities. They are slow, high-trust, and difficult to sell as a student-stage project.

## First Paid Offers

### One-Time Dispatch Audit

Price target:

```txt
INR 5,000-25,000 per site/report
```

Deliverable:

- usage analysis
- stress event summary
- degradation estimate
- dispatch comparison
- savings opportunity
- action points

### Monthly BESS Health Report

Price target:

```txt
INR 3,000-15,000 per month per site
```

Deliverable:

- monthly battery health trend
- degradation estimate
- risky operating periods
- net savings after degradation
- dispatch recommendation

### EPC Proposal Tool

For EPCs, the tool helps produce better solar+BESS proposals.

Possible pricing:

```txt
INR 10,000-50,000 setup
INR 2,000-10,000/month access
```

## Trust Language

Do not claim:

> This is 100% accurate battery degradation prediction.

Use this framing instead:

> This is a decision-support estimate based on battery operating stress, cycle usage, temperature exposure, and replacement cost assumptions.

Trust should be built through:

- clear assumptions
- transparent formulas
- sensitivity analysis
- strategy comparison
- confidence levels
- no black-box claims

## Website Positioning

Headline:

> Maximize BESS savings without destroying battery lifetime.

Subheadline:

> Upload site load, tariff, EV charging, PV, and battery telemetry data. BESS ProfitGuard estimates battery degradation cost and recommends a smarter charge/discharge schedule.

Primary actions:

- Run sample demo
- Upload CSV
- Request pilot analysis

Show three outputs:

- Battery health summary
- Revenue versus degradation comparison
- Optimized dispatch schedule

## What Not To Promise

Do not promise:

- automatic live BESS control
- guaranteed savings
- warranty-grade degradation accuracy
- safety-critical decisions
- grid-code compliance
- hardware integration readiness

Promise:

- analysis
- comparison
- recommendations
- transparent assumptions
- report generation
- better decision support

## Monetization Ladder

1. Free pilot to get real data and a case study.
2. Paid report to prove willingness to pay.
3. Monthly report for recurring revenue.
4. SaaS dashboard after repeated reporting demand exists.
5. API or white-label tool for EPCs and BESS service companies.

## Current Strategic Rule

Build toward SaaS, but sell the first version as:

> Degradation-aware battery operation report for solar+BESS, EV charging, and commercial sites.
